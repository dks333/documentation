## **Overview**

Understanding and optimizing the physical workplace environment is important to ensure both employee satisfaction and productivity. 

To help you better understand how meeting spaces are being used and to get better insights into the environmental conditions of these spaces, Neat devices come equipped with a wide range of different sensors. 

Through these sensors, you can gather rich, granular data that can help you paint an accurate picture of  both room utilization and physical conditions, which you can then use to optimize the workplace according to your desired outcome. 

## [**​**](https://neat-f9261ca9-welcome-page.mintlify.app/webapi/sensors#introduction-to-sensor-data)

**Introduction to sensor data**

Sensor data is being sampled on the device every second, and pushed to the Neat Pulse cloud every 5 minutes. The only exception is **people count** which is also being pushed to the cloud within 10 seconds if the occupancy changes. Learn more about or API rate limits [**here**](https://neat-f9261ca9-welcome-page.mintlify.app/webapi/webapi/ratelimits) to optimize the way you fetch this data. 

Sensor data is available at two different levels: \_device level \_and _room level_. The device level returns sensor data specifically gather by that device (bar, bar, board etc), whereas the room level returns an aggregate reading using data from all devices in that room. We generally recommend that you used the room level as it provides a more accurate representation of the overall room conditions. 

Sensor data is available on both the Neat Pulse management platform, and through the Neat Pulse web API ([**see documentation here**](https://pulse.neat.no/apidocs/#api)). The data is always in syn between the two. 

Note that all Neat devices have the same set of sensors - please consult the table below for an overview. 

Please also see [**this support article**](https://support.neat.no/article/neat-sense-and-neat-sensors/) for more details on our sensors. 

## [**​**](https://neat-f9261ca9-welcome-page.mintlify.app/webapi/sensors#data-storage)

**Data storage**

No sensor data is being stored on the device. If either the device or the Neat Pulse cloud is unavailable, samples are discarded. 

The Neat Pulse cloud currently does not store or aggregate any sensor data. Instead, we encourage each API customer to retrieve and store data at an interval and in a format that best suits each individual use case.

## [**​**](https://neat-f9261ca9-welcome-page.mintlify.app/webapi/sensors#retrieving-sensor-data-using-api)

**Retrieving sensor data using API**

There’s currently three ways to retrieve sensor information using the Neat Pulse web API:

- [**GET endpointsSensorData**](https://pulse.neat.no/apidocs/#api-Endpoints-endpointsSensorData). Retrieve sensor data for an individual device, using an individual ID as a unique identifier
- [**GET endpointsBulkSensorData**](https://pulse.neat.no/apidocs/#api-Endpoints-endpointsBulkSensorData). Retrieve sensor data for all devices enrolled in your Neat Pulse tenant in one call, using a Pulse org ID as a unique identifier
- [**GET roomsGetSensorData**](https://pulse.neat.no/apidocs/#api-Rooms-roomsGetSensorData). Retrieve sensor at the room level, using a room ID as the unique identifier. _This is the preferred method of retrieving sensor information as it combines the readings of all devices in the same room and this provides a more accurate representation of the room conditions._
